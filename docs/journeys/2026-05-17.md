# Bollinger Squeeze Strategy Rewrite Journey (2026-05-17)

## Feature and Why
Rebuilt the Bollinger backtest around the new squeeze strategy so the engine follows the latest documented rules directly, instead of layering more conditions on top of the older mean-reversion logic.

## What Was Implemented
- Replaced Bollinger backtest config with squeeze-focused fields:
  - `setupWindowDays`, `volumeMultiplier`, `breakEvenProfitPct`, `maxHoldDays`.
- Rewrote backend entry path in `BollingerBacktestService`:
  - 60-day squeeze arm (`bbSqueeze`),
  - breakout trigger (`close > bbUpper`),
  - volume confirmation (`volumeRatio20 >= volumeMultiplier`),
  - entry at close.
- Rewrote backend exit path to 3-phase roadmap:
  - Phase 1 structural stop,
  - Phase 2 break-even at configured profit threshold,
  - Phase 3 staircase trail using previous day low,
  - intraday stop simulation from daily candle (`open` gap / `low` touch).
- Updated Bollinger contracts/UI:
  - `frontend/src/types.ts`,
  - `frontend/src/pages/BollingerBacktestPage.tsx`,
  - `frontend/src/pages/BollingerBacktestPage.test.tsx`.

## Key Decisions and Tradeoffs
- Kept trend alignment (`SMA200`) informational in criteria/debug (not a hard backtest filter), matching current strategy doc status.
- Used conservative daily-candle sequencing for stop simulation: stop checks run before same-day phase upgrades to avoid optimistic bias.
- Preserved existing endpoint and page flow to minimize integration churn while changing strategy behavior.

## Hotfix Update (Sparse Listing Handling)
- Issue observed: `/api/strategy/bollinger/backtest` could fail hard when a newly listed stock (for example `FRACTAL`) had sparse/unreliable history during sync/indicator build.

## Fast Day-1 Breakout Update

### Why
- User requirement: do not wait for day-2 confirmation when day-1 breakout is already exceptionally strong in both price and volume.

### What Was Implemented
- Added a fast day-1 entry override in squeeze backtest:
  - `close(today) > bbUpper(today)`
  - `close(today) vs close(yesterday) >= 8%`
  - `volume(today) / SMA20(volume, excluding today) >= 10x`
- Kept existing 2-day breakout + volume flow unchanged as fallback for non-exceptional candles.
- Added computation for `volume(today) / previous-20-day average` (excluding current day) inside squeeze state build.
- Updated docs:
  - `docs/strategies/bollinger-squeeze-strategy.md`
  - `docs/strategies/bollinger-backtest-source-of-truth.md`

### Validation and Outcome
- Rebuilt and restarted service.
- Re-ran SAREGAMA squeeze backtest:
  - Entry shifted from `2026-05-15` to `2026-05-14`.
  - Entry signal now: `SQUEEZE_BREAKOUT_FAST_DAY1`.
  - Entry reasoning confirms:
    - price breakout `14.77% >= 8%`
    - volume vs previous-20-day average `105.54x >= 10x`.

### What Was Implemented
- Backend (`BollingerSqueezeService`):
  - corrected `maxRsi52w` / `maxRsi1y` to use `HighestValueIndicator(RSI, 252)`.
  - separated `todayRsi` from yearly-max RSI.
  - guarded drawdown divide-by-zero and invalid tracker inputs (`buyPrice <= 0`, missing buy-date candle).
  - removed unused dependencies from service constructor and DI provider.
- Frontend (squeeze components/types):
  - added missing squeeze contracts in `frontend/src/types.ts`.
  - removed `any` escapes and added typed tracker row/form models.
  - prevented delete call when row has missing `tradeId`.

## Trend Since Filter1 Origin (Close vs Previous Close)

### Why
- Needed a raw directional context for stocks where Filter1 has hit but Filter2 is pending.
- Goal: visualize how price has behaved from first squeeze hit to today without status abstraction.

### What Was Implemented
- Backend (`BollingerSqueezeService`) now computes trend from `Filter1 Origin Date` to latest bar using `Close(t) vs Close(t-1)`.
- Added new response fields:
  - `trendPatternFromFilter1` (run-length pattern, e.g. `U3-D1-U3-D3`)
  - `trendOverallFromFilter1` (`UPTREND` / `DOWNTREND` / `SIDEWAYS` / `UNKNOWN`)
  - `trendNetMovePctFromFilter1` (net % move from Filter1 origin close to current close)
- Frontend:
  - Added `Trend Since F1` column with pattern + overall + net move.
  - Included trend fields in raw CSV export.

### Validation and Outcome
- `mvn -q -pl core,resources,service,cron-job -DskipTests compile` passed.
- `npm --prefix frontend run -s build` passed.
