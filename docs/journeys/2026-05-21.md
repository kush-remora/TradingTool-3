# Feature Journey - 52-Week High Backtest (2026-05-21)

## Feature and Intent
Built a fast backtest workflow to test a breakout rule:
- detect first eligible 52-week-high breakout,
- buy at next-day open,
- exit at +20%,
- keep open if target is not hit,
- enforce 90-day cooldown for re-entry.

This was built to quickly validate strategy behavior over recent history with minimal setup overhead.

## What Was Implemented
- Backend strategy package:
  - `core/.../fiftytwohigh/FiftyTwoWeekHighBacktestModels.kt`
  - `core/.../fiftytwohigh/FiftyTwoWeekHighBacktestService.kt`
- API additions in `StrategyResource`:
  - `GET /api/strategy/52-week-high/universes`
  - `POST /api/strategy/52-week-high/backtest`
  - request validation helper for this strategy.
- Frontend additions:
  - page: `frontend/src/pages/FiftyTwoWeekHighBacktestPage.tsx`
  - hook: `frontend/src/hooks/use52WeekHighBacktest.ts`
  - route/menu wiring in `frontend/src/App.tsx`
  - types in `frontend/src/types.ts`

## Key Decisions and Tradeoffs
- Reused existing `index_constituents` source for universe options and symbol resolution.
- Kept manual symbols as optional filter over selected indices for predictable token resolution.
- Used simple bucket classification (`LARGE`/`MID`/`SMALL`/`OTHER`) from selected memberships.
- Prioritized minimal, readable implementation over config-heavy flexibility.
