# Feature Journey - NSE `-IV` Instrument Token Fallback

## Feature Name and Why It Was Built
- Feature: Instrument token resolver support for NSE symbols where Kite uses `-IV` suffix.
- Why: Symbols like `PGINVIT` were not resolving because Kite stores the tradingsymbol as `PGINVIT-IV`.

## What Was Implemented
- Extended `InstrumentTokenResolverService` NSE fallback logic to try a suffix list instead of only `-BE`.
- Added `-IV` to the NSE fallback suffix list.
- Updated expected-key diagnostics to include all configured NSE fallback keys.
- Added unit test coverage for `PGINVIT -> PGINVIT-IV -> 894465`.

## Key Decisions and Tradeoffs
- Chose a minimal list-based fallback (`-BE`, `-IV`) to keep behavior explicit and readable.
- Avoided broader fuzzy matching so we do not accidentally resolve to incorrect NSE variants.

## Validation Run and Outcomes
- Ran: `mvn -pl core -Dtest=KiteInstrumentTokenResolverTest test`
- Outcome: Passed.

## Next Follow-ups
- If more consistent NSE suffix variants appear in production logs, add them explicitly to the fallback list with tests.
