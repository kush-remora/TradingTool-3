## Live market compact widget first pass

### Feature and why it was built

Started the first production implementation of the shared live-market compact widget so table screens can show higher-value intraday context without trying to fit the full O/H/L/V block everywhere.

### What was implemented

- Confirmed the local Kite SDK contract exposes:
  - WebSocket tick `averageTradePrice`
  - quote snapshot `averagePrice`
- Extended the shared backend live-market contract with `averagePrice` across:
  - `TickSnapshot`
  - `LiveMarketUpdate`
  - `StockQuoteSnapshot`
- Updated the live SSE path and quote snapshot path to pass `averagePrice` through instead of dropping it.
- Added the first compact `standard` mode to `LiveMarketWidget`.
- Kept a `wide` mode path available for richer layouts later.
- The current default shared cell now shows:
  - `LTP`
  - `% change`
  - `Above Avg` / `Below Avg`
  - `Vol x T-1`
- Replaced the temporary `B / S / N` pressure badge with a Groww-style split pressure bar using live buy/sell quantities.
- `standard` mode now shows a micro pressure bar.
- `wide` mode now shows:
  - `Buy xx% (qty)`
  - `Sell xx% (qty)`
  - split pressure bar
- Applied the first explicit `wide` mode rollout to:
  - `Phase D`
  - `Trade Book`
- Kept the broader scanner tables on `standard` mode.
- In `wide` mode, kept `Avg` as a numeric field and used color on the value itself instead of adding `Above Avg` / `Below Avg` text.
- Added subtle `T-1` coloring to improve scan speed:
  - green when `>= 2.0x`
  - amber when `1.3x` to `< 2.0x`
  - neutral when `< 1.3x`
- Added the first shared clickable expand flow:
  - clicking the live widget opens a right-side detail drawer
  - drawer shows richer live context plus recent market history
- Added the missing backend stock-detail endpoint used by the drawer history panel:
  - `/api/stocks/by-symbol/{symbol}/detail`
- URL-encoded frontend stock-detail symbol requests so path-based symbols stay safe.
- Extended the stock-detail response for drawer analytics with:
  - `avg_volume_20d`
  - `pivot_levels`
- Updated the live detail drawer to:
  - fall back to stock-detail `20D Avg Vol` when live data does not provide it
  - show a new `Support and Resistance` card using classic pivot levels
- Reworked the recent-session history table inside the drawer to be more readable in a narrow layout:
  - `Date` now includes weekday in the same cell
  - `Open` and `Close` are merged into a single `Session` column with colored `close vs open` context
  - `Low` and `High` are merged into a single `Range` column with percent and point movement
  - `Volume` is kept visible with `vs 20D` context

### Key decisions and tradeoffs

- Used the real Kite `averageTradePrice`/`averagePrice` fields instead of deriving a proxy in the frontend.
- Kept the rollout intentionally small and plugin-safe: no intraday panel yet, and the buy/sell pressure bar stays secondary rather than becoming a primary signal.
- Left `wide` mode available but did not force every table to adopt a larger widget immediately.
- Chose `wide` only for conviction-oriented tables where row count is lower and each row already deserves more live context.
- For `wide` mode, preferred numeric comparison over explanatory text so users can read `LTP` vs `Avg` faster with less visual noise.
- Kept `T-1` color thresholds simple and explicit so the cue helps scanning without adding another interpretive label.
- Reused the existing stock-detail history panel inside the drawer so the first expand flow stays simple and frontend-only.
- QA exposed that the frontend history hook already expected a stock-detail endpoint that was not yet implemented in `StockResource`; adding that route fixed the `HTTP 404 Not Found` drawer error.
- For support/resistance, chose classic pivot points from our own latest completed daily candle instead of adding a broader indicator engine or copying another platform's private logic.
- Kept the `20D Avg Vol` fix as a drawer fallback only, which solves the real UX gap without expanding the live SSE contract.
- For the session history table, chose column consolidation over adding more raw fields, because the drawer width rewards interpretation-first presentation more than spreadsheet-style decomposition.
- Replaced the letter badge with a split bar because the bar preserves strength and balance much better than collapsing `56/44` and `90/10` into the same single-letter outcome.
- Added formatted raw quantity beside the wide-mode percentages so pressure strength is visible both as balance and as actual order-book size.
- Kept the pressure bar intentionally low-emphasis so it supports price and volume context rather than overriding it.

## Live market listener cleanup

### Feature and why it was built

Followed up on the live-market rollout after runtime profiling showed the app still felt heavy on large live tables even after the backend stream lifecycle fix.

### What was implemented

- Stopped mounting the live detail drawer until the user actually opens it.
- Removed the drawer's duplicate live subscription and fed it from the already-mounted widget's live snapshot instead.
- Reworked the shared live-market hook manager so it dispatches updates only to listeners for the matching symbol.
- Removed the unused `buyerDominancePct` query parameter from the SSE URL builder because the backend ignores it and it could still force needless reconnect churn.
- Updated the shared quote snapshot hook so it does not perform an initial `/api/stocks/quotes` fetch after market close and stops its polling loop when market hours end.
- Added focused tests for:
  - drawer mount deferral
  - drawer fallback rendering with injected live data
  - symbol-scoped live update dispatch in the shared hook

### Key decisions and tradeoffs

- Kept the fix frontend-only because the remaining cost was in listener fan-out and duplicate subscription work, not in a new backend contract issue.
- Preferred removing duplicate subscriptions over adding memoization or more React-level caching because the extra work was structural, not a rendering micro-optimization problem.
- Passed live data into the drawer from the widget instead of introducing a more global live-market context, which keeps the diff smaller and easier to reason about.

### Validation run and outcomes

- `npm --prefix frontend test -- --run src/components/LiveMarketWidget.test.tsx src/components/LiveMarketDetailDrawer.test.tsx src/hooks/useLiveMarketData.test.tsx src/hooks/useStockQuotes.test.tsx` passed.
- `npm --prefix frontend run build` passed.
- Manual smoke check with the backend running locally and Phase D open after market close showed only the dashboard load requests and no follow-up `/api/stocks/quotes` or `/api/market/live` traffic, which matches the market-hours gate.

### Next follow-ups

- If the machine still struggles during live market hours after this patch, profile the heaviest page with real market traffic and measure row count, per-tick React commits, and quote history panel cost separately.
