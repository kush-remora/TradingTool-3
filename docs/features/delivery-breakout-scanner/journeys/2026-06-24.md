# Delivery Breakout ETF Exclusion

## Feature and purpose

Removed ETF symbols from the Delivery Breakout Validation screen so the ranked table reflects stock breakout candidates only.

## Implemented

- Added a small backend ETF loader under `core/.../strategy/deliverybreakout/DeliveryBreakoutEtfService.kt`.
- Wired the Delivery Breakout scanner to read `manual-input/eq_etfseclist.csv` during dashboard assembly.
- Excluded any row whose normalized symbol appears in that ETF CSV before liquidity checks and ranking.
- Added focused tests for ETF CSV parsing and filtering behavior.

## Key decisions and tradeoffs

- Kept the ETF source local and file-based because Kush already added the NSE ETF CSV and the scanner only needs a simple yes/no exclusion.
- Avoided introducing a new database classification table because this fix is specific, small, and easier to maintain today.
- Chose fail-open behavior: if the ETF file is missing or malformed, the scanner still works and logs the issue instead of breaking the screen.

## Validation

- `mvn -f pom.xml -pl core -am test -Dtest=DeliveryBreakoutAnalyzerTest,DeliveryBreakoutEtfServiceTest -Dsurefire.failIfNoSpecifiedTests=false` passed.
- `mvn -f pom.xml -pl service -am -DskipTests compile` passed.
