# Groww Volume-Shocker Daily Ingestion

## Why

The useful downstream interpretation of Groww's daily volume-shocker list is not settled. The feature therefore preserves the complete top-100 daily observation set before adding shortlist or strategy rules.

## Implemented

- Added `groww_volume_shocker_daily` with dated rank, stock identity, mandatory Kite instrument token, price/volume inputs, 52-week bounds, market cap, and an approximate market-cap category.
- Kept the canonical `core/src/main/resources/tables.sql` schema aligned with the standalone migration.
- Added strict parsing for `manual-input/groww_volume_shocker`; stripped API responses, incomplete lists, duplicate stocks, and invalid values are rejected.
- Added a manual Kotlin job that assigns the current `Asia/Kolkata` date.
- Resolve every Kite token before the database transaction; the date is replaced atomically only after all 100 rows pass validation.
- Resolve BSE-only Groww rows by their numeric BSE scrip code through Kite's exchange-token mapping.
- Resolve Groww NSE symbols against Kite's `-BE`, `-IV`, and `-SM` segment suffixes, including `KODYTECH` to `KODYTECH-SM`.
- Applied internal market-cap thresholds: `LARGE >= 105000`, `MID >= 34700 and < 105000`, and `SMALL < 34700` crore.
- Kept all domain validation in the ingestion service; SQL uses types, required columns, defaults, and a generated technical primary key without business `CHECK` or `UNIQUE` constraints.

## Decisions and Trade-offs

- Direct Groww API automation was rejected for V1 because the public response strips company and exchange identity fields needed for token resolution.
- Calculated price-change and volume-shock ratios are not stored; their raw inputs remain available for one later shared definition.
- Market-cap categories are rough TradingTool context, not official AMFI classifications.

## Validation

- Focused core tests cover parsing, business uniqueness, market-cap boundaries, and BSE exchange-token resolution.
- Full Maven dependency chain through `cron-job`: compiled successfully.
- Full core suite has one unrelated pre-existing failure: `GrowwWatchlistSyncServiceTest` expects `groww`, while the current user-owned default is `groww_HIGH_QUALITY`.

## Manual Run

After applying `db/migrations/20260622_groww_volume_shocker_daily.sql` and updating the input file:

```bash
mvn -pl cron-job -am install -DskipTests
mvn -f cron-job/pom.xml exec:java \
  -Dexec.mainClass=com.tradingtool.cron.GrowwVolumeShockerIngestionJobKt
```
