# Volume Shocker Dashboard V1

## Feature and purpose

Built the first review dashboard on top of `groww_volume_shocker_daily` so the daily top-100 volume-shocker list can be studied as an event set rather than just stored. The immediate goal is faster Wyckoff research: inspect one ingested date, see repeat names, compare delivery behavior before the event, and hand off interesting names into the existing Phase 1 workflow.

## Implemented

- Added `/api/strategy/volume-shocker/dates`, `/api/strategy/volume-shocker/dashboard`, and `/api/strategy/volume-shocker/detail`.
- Extended the Groww volume-shocker read DAO with date listing, date snapshot reads, symbol detail lookup, and multi-date repeat lookup.
- Added a dashboard service that enriches each stored top-100 row with:
  - event-day delivery volume and delivery %
  - 10-trading-day appearance count
  - 10-trading-day consecutive streak length
  - 200 SMA price and distance from 200 SMA
  - 10-day max pre-event delivery volume
  - event-day delivery vs max pre-event delivery ratio
  - `pre_event_accumulation_hint`
  - `WYCKOFF_PHASE_D` vs `NO_TAG`
- Added a new React console page for the volume-shocker dashboard with:
  - ingested-date selector
  - quick toggles for `show only pre-event accumulation hint = yes` and `show only repeat names`
  - full column-filterable table
  - inline expandable 15-day detail view
  - single-stock and bulk Phase 1 handoff actions
- Extended the existing Phase 1 page to accept prefilled symbols and an `asOfDate` from query params while still requiring a manual run click.

## Key decisions and tradeoffs

- Kept the dashboard observation-first. No watchlist workflow, labeling workflow, or execution workflow is included in v1.
- Used the existing Phase 1 scanner as the first `WYCKOFF_PHASE_D` tag source instead of inventing a second accumulation engine.
- Kept the inline row detail lazy-loaded so the main table stays lighter and the detail API only runs for rows the user actually expands.
- Treated delivery percentage as a display/context field only in v1. The first accumulation-hint rule is based on absolute delivery-volume behavior plus quiet price.
- Chose a simple explicit accumulation-hint assumption for v1: the strongest pre-event delivery day in the prior 10 trading sessions must close between `-2%` and `+2%` and its delivery volume must be at least `2x` the average of the other candidate pre-event delivery days.
