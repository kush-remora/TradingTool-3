# Feature Journey - 104W Live Strategy

## Feature and Why
- Feature: 104-week live breakout screener.
- Why: convert the 104-week backtest into a manual daily scanner that helps pre-position attention (`Near Breakout`) and action completed (`Hit in Last 2 Weeks`, `Hit Today`).

## What Was Implemented
- Backend live strategy service with:
  - 504-trading-session prior-high breakout baseline.
  - Breakout hit rule: `today high >= prior 104-week high`.
  - Near threshold: within 5% below breakout level.
  - Buckets: near breakout, hit in last 10 trading sessions, and hit today (last 2 trading sessions).
  - Cooldown: 180 trading sessions (no re-hit during cooldown).
  - No overlap from near to recent-hit/cooldown names.
- Universe support:
  - `WATCHLIST` from `stocks` table (NSE rows).
  - `INDEX:<index_key>` from `public.index_constituents`.
  - Multi-select across both.
- API endpoints:
  - `GET /api/strategy/52-week-high/live/universes`
  - `POST /api/strategy/52-week-high/live/run`
  - `POST /api/strategy/52-week-high/live/telegram`
- Frontend:
  - New dedicated `104W Live` page.
  - Manual run controls, sortable/filterable tables per bucket.
  - One-click per-row Telegram send.

## Key Decisions and Tradeoffs
- Reused existing 52W strategy package to keep behavior locality and reduce cognitive overhead.
- Kept near-bucket exclusion from recent-hit/cooldown names to avoid noisy duplicate actionables.
- Kept Telegram send as per-row API call rather than bulk-send for precise operator control.

## Validation and Outcomes
- `mvn -q -pl core,resources,service -DskipTests compile` passed.
- `npm --prefix frontend run -s build` passed.

## Next Follow-Ups
- Add focused backend tests for:
  - cooldown boundary,
  - hit window boundary (10 sessions),
  - hit-today boundary (2 sessions),
  - near-threshold edge at exactly 5.00%.
