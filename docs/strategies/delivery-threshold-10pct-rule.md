# Delivery Threshold 10% Profit Rule (Working Draft)

## Objective
Measure:
- how many buy entries are generated by delivery-threshold signals, and
- how long it generally takes each entry to reach +10% profit.

## Market-Cap Buckets and Delivery Thresholds
- Large/Mid Cap: delivery % > 55
- Small Cap: delivery % > 70
- Micro Cap: delivery % > 85
- Nano Cap: delivery % > 92 (optionally tighten to 95 in later experiments)

## Backtest Config (Current)
- `thresholds`: set by `index_key` / cap group.
- `profit_pct`: default `10`.
- `date_range`: default last `1 year`.
- `entry_mode`: removed for now (fixed behavior is always next-day open entry).

## Entry Rule
For each stock:
- If delivery % crosses its bucket threshold on day T, create one buy entry on day T+1 open.
- If threshold is hit on consecutive days, create separate buy entries for each day.
- Each signal is treated as an independent position.

## Exit Rule
- Exit only when price first reaches +10% from that position's entry price.
- No stop loss.
- Drawdowns are ignored.
- If +10% is never reached in available data, the position remains open forever (open position).

## Behavior Clarifications
- A position can go down -10%, -20%, etc. and still be held.
- If it later reaches +10%, that later date is the exit date.
- We care about the first date +10% is hit and days taken from entry to that date.

## Required Output Metrics
- Total Signals / Total Buys
- Hit Count (+10% reached)
- Hit Rate = Hit Count / Total Buys
- Days to +10% (average, median, minimum, maximum)
- Open Position Count (never reached +10%)
- Open Position Age (days since entry to dataset end)

## Scope Note
This model is intentionally a raw signal-efficiency test, not a risk-managed portfolio strategy.
