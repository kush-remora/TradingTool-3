# 2026-05-16 Implementation Journal

## Feature
Wyckoff Launch v1: Weekly Index Membership Cron (Supabase + Kite Token Mapping)

## Why
Create an index-membership flow independent from watchlist/stocks table so index universe data can be synced weekly and used as a separate source of truth.

## What Was Implemented
- Added weekly cron workflow:
  - `.github/workflows/index-constituent-weekly-sync.yml`
- Added config-driven index sync input:
  - `wyckoff-market-cycle/config/index_sync_config.json` (v1 includes only Nifty 50)
- Added isolated SQL contract:
  - `wyckoff-market-cycle/sql/index_constituents.sql`
- Added core sync pipeline:
  - CSV fetch/parse with strict Symbol-header validation
  - row normalization + dedupe
  - Kite instrument token resolution
  - batched upsert
  - soft deactivate missing symbols per index
- Added artifact outputs:
  - `build/reports/index-constituent-sync/latest.json`
  - `build/reports/index-constituent-sync/latest.md`

## Corrections Applied After Feedback
- Removed incorrect assumption that `stocks` table is source of truth for this flow.
- Updated design to store CSV fields in `index_constituents` directly:
  - `company_name`, `industry`, `series`, `isin_code`
- Simplified cron entrypoint:
  - removed CLI args and config-path override
  - now always loads module config file

## Key Decisions
- Keep this flow separated from watchlist data model.
- Keep one normalized index relation table for many-to-many stock/index mapping.
- Keep removals as soft deactivate (`is_active=false`) for traceability.

## Validation
- `mvn -q -pl core,cron-job -DskipTests compile` passed.
- Focused tests for CSV parser and sync service passed.
- Note: intermittent Kotlin daemon EOF occurred once during test compile; rerun passed.

## Follow-ups
- Add additional indices in `index_sync_config.json` when required.
- Optionally add DB migration script for existing Supabase environments where table already exists without the new metadata columns.
