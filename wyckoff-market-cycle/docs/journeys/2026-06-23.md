# SMA Buy Zone Screener

## Feature and purpose

Added a simple read-only SMA buy-zone screener so Kush can manually run an existing universe and quickly spot names drifting back toward `SMA200` without missing them in a larger watchlist.

## Implemented

- Reused the existing `hotsma` backend flow and expanded it to return every stock in the selected universe instead of only signal hits.
- Added `SMA100`, `SMA200`, `% from SMA100`, `% from SMA200`, absolute `% to SMA200`, `RSI 14`, drawdown from `20D` and `60D` highs, consecutive red days, and `3D move %`.
- Added a simple status model where `BUY_ZONE` means price is at or below `SMA200 + 5%`.
- Added `GET /api/strategy/hot-sma/universes` and `POST /api/strategy/hot-sma/run`.
- Added a new console page and sidebar entry: `SMA Buy Zone`.

## Decisions and tradeoffs

- Kept the rule intentionally narrow: `SMA200` drives the buy-zone status; `SMA100` is context only.
- Returned the full table for the selected universe instead of hiding non-qualifying names, because the user still wants the wider pullback picture.
- Kept the screen read-only and skipped alerts, notes, and backtesting to preserve the daily scanner workflow.
