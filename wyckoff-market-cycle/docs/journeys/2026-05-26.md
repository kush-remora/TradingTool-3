# Wyckoff Phase-1 Scanner (2026-05-26)

## Feature and Why
Built a dedicated Wyckoff Phase-1 scanner flow so we can run accumulation detection directly (latest matched signal in last 5 sessions), without altering the older delivery-threshold backtest workflow.

## What Was Implemented
- New Phase-1 backend API surface:
  - `GET /api/strategy/wyckoff/phase1/config`
  - `GET /api/strategy/wyckoff/phase1/columns`
  - `POST /api/strategy/wyckoff/phase1/run`
- New core Phase-1 module for:
  - request/config/response models
  - config loading (`wyckoff_phase1_config.json`, `wyckoff_phase1_table_columns.json`)
  - scanner engine (delivery pass, density, z-score, absorption, ROC-20 range, DMA-200 range, passed_count, run-length)
  - universe resolution (indices + watchlist + manual symbols union)
- New frontend page `Wyckoff Phase-1` with:
  - universe multiselect from `/api/screener/universes`
  - optional manual symbols
  - run guard (must select at least one universe key)
  - table columns auto-driven by column config file
- Existing Delivery Threshold Backtest flow kept unchanged.

## Key Decisions and Tradeoffs
- Kept this as a separate page/API to avoid breaking existing backtest behavior.
- `low_volume_high_delivery_info` remains informational only (no ranking logic).
- Used available candles for SMA window when history < 200 days to handle IPO/new listings safely.
- Included `days_ago` and `delivery_threshold_pct` in payload for optional future UI use, but left disabled in default column config.

## LVQ_DQ Enhancement (same day update)
- Added configurable LVQ_DQ logic:
  - `rollingMinDays` (default 63)
  - `nearMinPctOfRollingMin` (default 95.0)
  - `lookbackDays` (default 15)
  - `requireDeliveryPass` (default true)
- LVQ hit now requires both:
  - near-rolling-min volume condition
  - same-day delivery threshold pass
- Added row fields:
  - `lvq_dq_pass`
  - `lvq_hit_count_15d`
- `passed_count` updated from 6 to 7 flags by including LVQ pass.
- Row inclusion logic updated to:
  - include when `delivery_pass = 1 OR zscore_pass = 1`.

## Signal Lookback Config Update
- Added `signalLookbackDays` to Phase-1 config (default `10`).
- Scanner now uses this value instead of hardcoded 5-day window to find latest recent signals.
- This helps recover recently-ended accumulation phases when scans are run intermittently.

## Fast Scan Performance Update
- Optimized Phase-1 runtime path for 100-200 symbol scans:
  - batched delivery read for all instrument tokens in one DAO query.
  - parallel candle loading with bounded concurrency (`runtime.maxParallelSymbols`).
  - dynamic warmup range calculation (instead of fixed 500-day fetch window).
- Added runtime config knobs in `wyckoff_phase1_config.json`:
  - `runtime.enableCandleBackfill` (default `false`) to avoid slow interactive backfill calls.
  - `runtime.maxParallelSymbols` (default `16`) to tune throughput vs DB pressure.
- Preserved signal-generation logic and response contract; changes are load-path only.
