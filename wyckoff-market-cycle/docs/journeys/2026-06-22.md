# Index Constituents Stock Maintenance

## Feature and purpose

Consolidated stock-universe maintenance into `public.index_constituents` so index syncs and the Groww watchlist share one explicit source of truth. The legacy `stocks` data is intentionally discarded.

## Implemented

- Added the canonical `groww` membership key and routed Groww file/upload ingestion to it.
- Deactivate Groww rows removed from the latest import; reject empty parsed imports before changing existing membership.
- Removed the `stocks` table contract, stock CRUD remnants, and `stock_id` from delivery data and reports.
- Routed WATCHLIST resolution, delivery universe loading, universe counts, and ticker subscriptions through active `index_constituents` rows.
- Added an idempotent PostgreSQL migration at `db/migrations/20260622_index_constituents_stock_source.sql`.

## Decisions and tradeoffs

- Existing `stocks` rows, notes, priorities, and tags are not migrated by explicit owner decision.
- Kept `/api/stocks/quotes` and `/api/stocks/instruments`; these are market-data endpoints and do not depend on the removed table.
- Kept the composite `(index_key, symbol)` identity so one symbol can belong to multiple index/watchlist universes.
