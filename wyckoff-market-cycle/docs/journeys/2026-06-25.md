## Phase D daily-market-field wording fix

### Feature and why it was built

Clarified a misleading Phase D review column after manual validation showed the UI was calling the latest completed daily close `Prev Close`, which made correct backend values look broken.

### What was implemented

- Renamed the Phase D table column from `Prev Close / Vol` to `Daily Close / Day Vol`.
- Labeled the grouped cell lines as `Day %` and `Day Vol` so they read as daily snapshot fields instead of live-market or previous-close fields.
- Added a separate `T-1 Close / T-1 Vol` column without backend changes.
- Derived `T-1 Close` from the existing latest daily close plus daily `% change`, and used the stored `previousDayVolume` as `T-1 Vol`.
- Added a focused frontend assertion to lock in the clearer wording.

### Key decisions and tradeoffs

- Kept the backend data contract unchanged because it was already correct: `closePrice` is latest daily close, `pctChange` is day-over-day change, `volume` is latest daily volume, and `previousDayVolume` remains the separate `T-1` field used by Wake-Up.
- Chose the smallest truthful UI fix instead of introducing new fields or duplicating daily context through storage changes.

### Validation run and outcomes

- `npm --prefix frontend run test:run -- src/pages/PhaseDScannerPage.test.tsx` passed.
- `npm --prefix frontend run build` passed.

### Next follow-ups

- If more daily vs live confusion appears, consider a small tooltip on the column header that states these values come from the latest completed daily candle refresh.
