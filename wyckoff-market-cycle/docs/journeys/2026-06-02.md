# Feature Journey - Wyckoff Phase-1 Scanner (Universe-Only Selection)

## Feature Name and Why It Was Built
- Feature: remove symbol-source path from Wyckoff Phase-1 scanner controls.
- Why: scanner was defaulting to all watchlist symbols (~128), causing runs to ignore the intended universe-key-only flow and creating incorrect default scope.

## What Was Implemented
- Removed `Symbol Source (optional)` controls from `WyckoffPhase1Page`.
- Removed watchlist symbol fetch/state from the page.
- Updated scanner request build to send only selected `universeKeys` and strict-base-filter flag.
- Simplified local storage filter model for this page to:
  - `universeKeys`
  - `strictBaseFilter`
- Updated page tests for universe-only behavior and removal of symbol-source UI.
- Removed unused frontend type `WyckoffPhase1SymbolSourceMode`.

## Key Decisions and Tradeoffs
- Decision: keep backend request contract unchanged (`symbols` remains optional), but stop using it from this page.
- Tradeoff: symbol-only debug runs are no longer possible from this scanner UI, in favor of predictable universe-key behavior aligned with Phase-1 intent.
- Decision: preserve strict filter toggle behavior and persistence unchanged.

## Validation Run and Outcomes
- `npm --prefix frontend run test:run -- WyckoffPhase1Page` passed (4/4).
- `npm --prefix frontend run build` passed.
- Kotlin diff gate check run: no `.kt`/`.kts` files changed.

## Next Follow-ups
- Consider backend-side telemetry of `universeKeys` per run for auditability.
- Optionally remove now-unused symbol normalization in related non-Phase-1 screens if not needed.
