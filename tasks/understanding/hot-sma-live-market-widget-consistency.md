## Hot SMA live market widget consistency

The real requirement is broader than Hot SMA. Every screener should use one shared `Live Market` component with identical behavior in two modes: live SSE updates when available, and a non-live daily snapshot when live ticks are absent. In both modes the cell should show `ltp, % change, open, high, low, volume`, with the richer O/H/L/V block rendered directly inside the table cell instead of collapsing to price-only.

The implemented direction now matches that requirement more closely. `/api/stocks/quotes` was extended to return a shared snapshot shape including `change_percent`, the frontend quote hook now fetches snapshots even after market hours, and `LiveMarketWidget` now merges live SSE fields with the shared snapshot fields instead of depending on screener-specific fallback props. The `Live Market` columns on `HotSmaPage`, `PhaseDScannerPage`, `DeliveryBreakoutScannerPage`, `VolumeShockerDashboardPage`, `WyckoffPhase1Page`, and `TradeJournalTable` now pass the same quote snapshot type into the same shared widget path and sort by the same `% change` logic. Validation passed with focused Phase D and Hot SMA frontend tests, a frontend production build, and a `resources` module compile.

Follow-up expansion: Kush wants the existing Phase D `Wake-Up` volume-footprint behavior promoted into a global component and added to `SMA Buy Zone` because too many stocks cluster around `SMA200` without any effort/participation clue. The shared requirement is to compare current day volume versus `T-1` volume, use live volume while the market is open, and still show the same structured wake-up cell after hours from the latest market snapshot. The implemented solution adds `previous_day_volume` to `/api/stocks/quotes`, extracts a shared `WakeUpVolumeCell`, keeps the Phase D behavior on that shared path, and adds the same `Wake-Up` column to `HotSmaPage`. Focused Phase D and Hot SMA frontend tests passed again after the extraction, and the `resources` module compile still passed.
