# Phase 3 Spec vs Current Implementation

We reviewed two external Phase 3 documents from `/Users/kushbhardwaj/Downloads/wycoff/` against the current runtime code under `core/.../strategy/wyckoff/phase1`, the `/api/strategy/wyckoff/phase1/*` resource surface, and the existing Phase 1 frontend page. The current implementation is still a Phase 1 anomaly scanner with an optional strict base filter. It does not implement a dedicated Phase 3 or Remora-style footprint engine yet. The repo also already contains a newer replacement requirement at `docs/features/composite-volume-pipeline/2026-06-20/wyckoff-footprint-replacement-requirement.md`, and that internal requirement is stricter and better aligned than either downloaded spec for migration planning.

Validation outcome: there is useful overlap, but the downloaded specs should not be adopted literally. Good ideas worth carrying forward are the watchlist-only execution scope, explicit metric exposure, configurable market-cap parameters, volatility squeeze, and 200 SMA context gating. The parts that should not be adopted as-is are the older total-volume percentile-density version, the use of same-day data in the baseline, and the v3 use of mean basement delivery without a denominator floor. The repo’s newer replacement requirement already corrects these weaknesses by requiring a lagged basement, lagged delivery baseline, median delivery baseline, density, and explicit separation between footprint confirmation and breakout confirmation.
